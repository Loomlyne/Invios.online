   - Wrap in try/catch (fallback: no-op, button stays idle)

Create `src/components/documents/document-status-actions.tsx` as a SERVER component (no "use client" directive). Uses inline `"use server"` async functions in form action props, matching the existing pattern on detail pages.

Props interface:
```typescript
interface DocumentStatusActionsProps {
  kind: "invoice" | "quotation";
  id: string;
  status: string;
  convertedToInvoiceId?: string | null; // only for quotations
}
```

Implementation:

1. Import server actions at the top:
   ```typescript
   import { Send, Trash2, ArrowRight } from "lucide-react";
   import { setInvoiceStatusAction, deleteInvoiceAction } from "@/actions/invoices";
   import { setQuotationStatusAction, deleteQuotationAction, convertQuotationToInvoiceAction } from "@/actions/quotations";
   import { Button } from "@/components/ui/button";
   ```

2. Render a `<div className="grid gap-3 sm:grid-cols-2">` container.

3. **Invoice status transitions** (per D-03 and UI-SPEC):
   - `status === "draft"`: Show "Mark as sent" button (variant="accent", Send icon). Form action: `async () => { "use server"; await setInvoiceStatusAction(id, "sent"); }`
   - Always show "Delete invoice" button (variant="danger", Trash2 icon). Form action: `async () => { "use server"; await deleteInvoiceAction(id); }`

4. **Quotation status transitions** (per D-03 and UI-SPEC):
   - `status === "draft"`: Show "Mark as sent" (variant="accent", Send icon). Action: `setQuotationStatusAction(id, "sent")`
   - `status === "sent"`: Show "Mark as accepted" (variant="secondary"). Action: `setQuotationStatusAction(id, "accepted")`
   - `status === "sent"`: Show "Mark as rejected" (variant="secondary"). Action: `setQuotationStatusAction(id, "rejected")`
   - `status === "accepted"` AND `convertedToInvoiceId === null`: Show "Convert to invoice" (variant="accent", ArrowRight icon). Action: `convertQuotationToInvoiceAction(id)`
   - Always show "Delete quotation" (variant="danger", Trash2 icon). Action: `deleteQuotationAction(id)`

5. Each action button is a `<form>` wrapping a `<Button type="submit" className="w-full">`. Pattern:
   ```tsx
   <form action={async () => { "use server"; await actionFn(args); }}>
     <Button type="submit" variant={variant} className="w-full">
       {icon && <IconComponent className="size-4" />}
       {label}
     </Button>
   </form>
   ```

6. If no transitions are available for the current status (e.g., invoice "sent", quotation "expired"), only the delete button renders.

Export: `export function DocumentStatusActions({ kind, id, status, convertedToInvoiceId }: DocumentStatusActionsProps)`
