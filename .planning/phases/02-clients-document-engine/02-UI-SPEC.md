Inherited from Phase 1 (Tailwind 4px base unit). No new exceptions for Phase 2.

| Token | Value | Phase 2 usage |
|-------|-------|---------------|
| xs | 4px (`gap-1`) | Icon-to-label gaps inside badge |
| sm | 8px (`gap-2`, `p-2`) | Button cluster gaps (`flex gap-2`), badge row gaps |
| md | 16px (`gap-4`, `p-4`) | Form field grid gaps, info card grid gap, builder section gaps |
| lg | 24px (`gap-6`, `p-6`) | Card default padding, top-level page grid gap |
| xl | 32px (`gap-8`) | Builder split-view grid gap on xl |
| 2xl | 48px | — |
| 3xl | 64px | — |

Exceptions:
- Builder form card header: `px-5 py-5 sm:px-6` (20px / 24px) — matches Phase 1 onboarding panel pattern
- Info cards / doc summary rows: `px-4 py-4` (16px × 16px) — consistent with Phase 1 checklist items
- Client hero card content: `px-6 py-7 sm:px-8` (24px / 28px) — 28px is 4px-divisible, acceptable
- Status action button grid: `gap-3 sm:grid-cols-2` — 12px gap (multiple of 4)
- PageHeader back link: `w-fit` ghost button, no extra spacing

## Layout Structure

### List Pages (Clients / Invoices / Quotations)

```
PageHeader
  Title (display-text text-3xl sm:text-4xl font-semibold)
  StatStrip (total, active/draft, leads/sent)
  Actions: primary CTA button (variant="accent")

Optional create-inline Card (shown when ?create=1 or ?new=1 in URL)

Card (DataView container)
  CardHeader: DataViewToolbar (search, status filter, view switcher)
  CardContent: DataViewRenderer (list / kanban / table)
    EmptyState when no data
```

Default view modes per entity (Claude's Discretion — locked here):
- **Clients:** `list` (name-first, status badge, email meta)
- **Invoices:** `list` (number, client, total, due date, status badge)
- **Quotations:** `list` (number, client, total, expiry, status badge)

Rationale: list is the most scannable view for billing entities. Kanban is secondary for status-based pipeline workflows.

### Detail Pages (Invoice / Quotation)

```
Button variant="ghost" w-fit  ← Back to [list]

section  grid gap-5 xl:grid-cols-[0.95fr_1.05fr]
  ├── Card (left — detail + actions)
  │   CardHeader
  │     Badge variant="accent"  "Invoice detail" / "Quotation detail"
  │     DocumentStatusBadge
  │     CardTitle: document number
  │     CardDescription: client name • company
  │   Inline action buttons (flex flex-wrap gap-2):
  │     Edit → /edit   (variant="secondary" size="sm")
  │     PDF  → direct download  (variant="secondary" size="sm")
  │     Share → opens ShareModal  (variant="secondary" size="sm")
  │   CardContent
  │     Info card grid (gap-3 sm:grid-cols-2)
  │     Status action buttons (gap-3 sm:grid-cols-2)
  └── InvoicePreview (right — branded document, mode="page")
```

### Client Detail Page

```
section  grid gap-5 xl:grid-cols-[1.05fr_0.95fr]
  ├── Hero Card (dark background #17120F)
  │     Badge "Client detail" (border-white/12 bg-white/10)
  │     ClientStatusBadge
  │     h1 display-text text-4xl sm:text-5xl (client name)
  │     p text-sm leading-7 text-[#E9E0D5] (company • email • phone)
  │     Buttons: "New quotation" (accent), "New invoice" (secondary), "Edit client" (inverse)
  └── Metric cards grid (gap-4 sm:grid-cols-2 xl:grid-cols-1)

        MetricCard "Quotations" count + quoted total

section  grid gap-6 xl:grid-cols-[1.05fr_0.95fr]
  ├── Left column (grid gap-5)
  │     Card: Client profile (contact, tax, address info cards)
  │     Card: Recent invoices (doc summary rows or EmptyDocumentState)
  │     Card: Recent quotations (doc summary rows or EmptyDocumentState)
  └── Right Card: Actions / Edit form
        Badge variant="accent" ("Client actions" or "Edit client")
        ActionLinks or ClientForm depending on editMode
        Archive button (variant="danger" size="sm") in edit mode
```

### Document Builder (split-view)

```
grid gap-6 xl:grid-cols-[minmax(0,1fr)_480px]
├── Card (form — left)
│   CardHeader: Badge variant="accent" + CardTitle + CardDescription
│   Mobile-only: Dialog trigger Button variant="secondary" "Preview"
│   CardContent: form sections
│     "Document frame" — Client select, [status dropdown REMOVED per D-04 → read-only Badge]
│     "Dates" — issue/quotation date, due/expiry date
│     "Financials" — currency, tax rate, discount
│     "Line items" — structured rows (desc, qty, unit price) + Add / Remove
│     "Document settings" — language, invoice type (invoice only)
│     "Notes & terms" — textarea pair
│   Submit button variant="accent" (full-width on mobile, w-fit on sm+)
└── InvoicePreview (right — sticky, hidden below xl, mode="page")
    [mobile: Dialog content]
```

**D-04 implementation:** The status `<select>` in the builder is replaced with a read-only `DocumentStatusBadge` displayed in the CardHeader alongside the "Invoice builder" / "Quotation builder" badge. No status mutation from builder.

**D-05 implementation:** On save success, `router.push(result.redirectTo)` where `redirectTo` is the detail page URL.

## New Components for Phase 2

### ShareModal

**Trigger:** "Share" button on invoice/quotation detail page (`variant="secondary" size="sm"`, Share2 icon).

**Structure:**
```
Dialog
  DialogContent  max-w-md rounded-[1.6rem] p-6
    DialogHeader
      p text-sm uppercase tracking-[0.18em] text-muted  "Share"
      DialogTitle display-text text-2xl font-semibold  "Public link"
      DialogDescription text-sm leading-7 text-muted
        "Anyone with this link can view the document without signing in."
    URL display row  mt-4
      div rounded-[1rem] border border-border bg-[#FFF8EE] px-4 py-3
        p text-sm font-mono text-muted-strong truncate  {public URL}
    Copy button  mt-3 w-full
      variant="accent" when idle: "Copy link"  (Copy icon)
      variant="secondary" when copied (2s): "Copied"  (Check icon)
      Copy-success state: button briefly shows Check icon + "Copied" — no toast needed
    Secondary link  mt-3 text-center
      text-sm text-muted  "Open in new tab →"  (external link, text-accent-strong on hover)
```

**Copy-success feedback:** Button transitions from `variant="accent"` + "Copy link" to `variant="secondary"` + Check icon + "Copied" for 2000ms, then resets. No animation library — `setTimeout` reset. `prefers-reduced-motion`: skip the 2s reset delay, transition immediately.

**Source:** D-01 (share modal with copyable link), CONTEXT.md Specifics ("feel premium and on-brand — not a generic browser prompt").

### DocumentStatusActions

Contextual action button grid rendered on detail pages. Replaces free-form status dropdown per D-03.

**Invoice status transitions:**

| Current status | Available actions | Button variant |
|----------------|------------------|----------------|
| `draft` | "Mark as sent" (Send icon) | `accent` |
| `sent` | (no further invoice actions in Phase 2) | — |
| Any | "Delete invoice" (Trash2 icon) | `danger` |

**Quotation status transitions:**

| Current status | Available actions | Button variant |
|----------------|------------------|----------------|
| `draft` | "Mark as sent" (Send icon) | `accent` |
| `sent` | "Mark as accepted" | `secondary` |
| `sent` | "Mark as rejected" | `secondary` |
| `accepted` | "Convert to invoice" (ArrowRight icon) | `accent` |
| `expired` | (no transition — read-only) | — |
| Any | "Delete quotation" (Trash2 icon) | `danger` |

**Layout:** `grid gap-3 sm:grid-cols-2` — each action is a full-width `<form>` wrapping a `<Button className="w-full">`.

**Locked quotation treatment (D-07):** When a quotation status is `accepted` and a linked invoice exists, the Edit button in the detail header renders as disabled with a "Converted" badge overlay. Tooltip text: "This quotation has been converted into an invoice and can no longer be edited." Button: `disabled` attribute + `variant="secondary"` + opacity reduced (`opacity-50 cursor-not-allowed`).

**Source:** D-03, D-04, D-07 from CONTEXT.md.

### DocumentSummaryRow (client detail)

Reusable inline component for the "Recent invoices" and "Recent quotations" sections on the client detail page.

Link  href=/app/invoices/{id} or /app/quotations/{id}

**Source:** D-08 (simple summary cards, no DataView). Extends the existing pattern in `clients/[slug]/page.tsx`.

Difference from Phase 1 implementation: adds `DocumentStatusBadge` on the right (replacing the plain ArrowRight icon) so status is visible at a glance without opening the document.

### Destructive action confirmations

| Action | Approach | Copy |
|--------|----------|------|
| Delete invoice | Inline form submit (no modal) — irreversible | Button label: "Delete invoice". No confirmation dialog — the status action grid layout makes accidental tap unlikely. |
| Delete quotation | Inline form submit (no modal) — irreversible | Button label: "Delete quotation". Same rationale. |
| Archive client | Inline form submit in edit mode only | Button label: "Archive client". Edit mode is an explicit opt-in (`?edit=1`) that gates the archive button — provides natural friction. |

**Rationale for no confirmation modals:** The `variant="danger"` visual treatment, `size="sm"` physical separation from primary actions, and the explicit edit-mode gate (for archive) provide sufficient friction without a modal confirmation step. This is consistent with the CONTEXT.md principle of fast, intentional flows. If a future phase introduces bulk operations or financial cascades on delete, a confirmation modal should be reconsidered.
