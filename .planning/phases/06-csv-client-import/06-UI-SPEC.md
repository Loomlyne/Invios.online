## Spacing Scale

Standard 8-point scale — all values are multiples of 4. Pre-populated from existing `--space-*` fluid tokens and Tailwind usage patterns across the codebase.

| Token | Value | Usage in this phase |
|-------|-------|---------------------|
| xs | 4px | Icon gaps, inline badge padding |
| sm | 8px | Button icon gap, row cell padding |
| md | 16px | Wizard step content padding, form field gaps |
| lg | 24px | Sheet header padding, section separation |
| xl | 32px | Wizard step vertical breathing room |
| 2xl | 48px | Empty state vertical centering |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Touch targets (checkboxes in preview table): minimum 44px hit area via `size-11` class (matches existing `icon` button size)
- File drop zone: 96px min-height to communicate interactivity (`min-h-24`)

Source: existing `--space-section`, `--space-card`, `--space-grid` clamp tokens in `globals.css`; button `size-11` from `button.tsx`

## Typography

Pre-populated from `page-header.tsx`, `empty-state.tsx`, `button.tsx`, and `globals.css`. No new sizes introduced.

Maximum 2 weights used (400 body, 600 semibold). Label role uses weight 600 — visually distinct from Heading by size (14px vs 16px), not by weight.

| Role | Size | Weight | Line Height | Usage in this phase |
|------|------|--------|-------------|---------------------|
| Body | 14px (`text-sm`) | 400 (`font-normal`) | 1.75 (`leading-7`) | Row text in preview table, dropdown labels, description copy |
| Label | 14px (`text-sm`) | 600 (`font-semibold`) | 1.5 | Column mapping field labels ("Name", "Email"), form labels |
| Heading | 16px (`text-base`) | 600 (`font-semibold`) | 1.25 | Wizard step title, preview table column headers, result counts |
| Display | clamp(1.625rem → 2.25rem) | 600 (`font-semibold`) | 1.2 | Page `<h1>` "Clients" — unchanged from existing `PageHeader` |

Notes:
- Display heading uses `.display-text` class (Cormorant Garamond, `letter-spacing: -0.03em`) — already applied by `PageHeader`
- Error text in preview table: `text-sm` at weight 400, color `text-danger` (`#8D3D2E`)
- Monospace is not used; CSV column header names render in `text-sm font-semibold text-muted`

Source: `page-header.tsx` L25, `empty-state.tsx` L22–24, `button.tsx` L7

## Color

Pre-populated from `globals.css` CSS custom properties. 60/30/10 distribution confirmed by existing surface usage across the app.

Primary visual anchor on the clients page: the "Import CSV" accent button in `PageHeader.actions`.

| Role | Token | Hex | Usage |
|------|-------|-----|-------|
| Dominant (60%) | `--background` | `#f8f4ee` | Page background, wizard overlay backdrop |
| Secondary (30%) | `--surface` / `--surface-subtle` | `#fffdf9` / `#fdf9f4` | MobileSheet content area, preview table rows, card backgrounds |
| Accent (10%) | `--accent` | `#ca8a04` | Reserved list below |
| Success semantic | `--success` | `#24583a` | Valid row check indicator, success badge in result step |
| Destructive semantic | `--danger` | `#8d3d2e` | Invalid row highlight border, error text, error badge |

Accent (`#ca8a04`) reserved for:
1. "Import CSV" button in `PageHeader.actions` (variant=`accent`)
2. "Confirm import" primary CTA button in Step 3 Preview (variant=`accent`)
3. Focus ring on interactive controls (`--accent` via existing `:focus-visible` rule)
4. Download template link underline/hover state
5. Step indicator (current step): filled `bg-accent` circle

Accent is NOT used for: table row hover states, informational badges, checkbox checks.

Duplicate row warning: `--warning` variant Badge (`border-amber-700/20 bg-amber-50 text-amber-800`) — matches existing `badge.tsx` `warning` variant. Reserved for the per-row warning badge only.

Source: `globals.css`, `badge.tsx`, `button.tsx`

## Wizard Step Layouts

### Step 1: Upload

```
MobileSheetContent (max-w-2xl)
  MobileSheetHeader
    title: "Import clients"
    description: "Upload a CSV file to add clients in bulk."
    [X close button — ghost Button, aria-label="Close import wizard"]

  [File drop zone — dashed border, border-border, bg-surface-subtle, rounded-[var(--radius-md)], min-h-24]
    FileText icon (size-8, text-muted)
    "Drop your CSV here, or click to browse"   ← text-sm text-muted
    [hidden <input type="file" accept=".csv">]

  [Download template link — ghost Button with Download icon]
    "Download template"   ← text-sm

  [200-row cap warning — shown only when row count > 200]
    Badge variant=warning
    "This file contains {N} rows — only the first 200 will be imported."

  [Footer]
    Button ghost "Close" (aria-label="Close import wizard")
    Button accent "Next →"  ← disabled until file parsed successfully
```

### Step 2: Map Columns

```
MobileSheetContent
  MobileSheetHeader
    title: "Map columns"
    description: "Match your CSV columns to client fields."

  [Mapping grid — 2-column on desktop, 1-column on mobile]
    For each client field (name, company, email, phone, address, trn):
      Label "{Field name}"          ← text-sm font-semibold
      Select                        ← options: CSV headers + "Skip this field"
      [auto-mapped note if detected: text-xs text-muted "Auto-detected"]

  [Footer]
    Button secondary "← Back"
    Button accent "Next →"  ← disabled if "name" field is not mapped
```

### Step 3: Preview

```
MobileSheetContent (max-w-3xl for table legibility)
  MobileSheetHeader
    title: "Preview import"
    description: "{N} rows ready — review and uncheck any rows to skip."

  [Scrollable table — max-h-[50vh] overflow-y-auto]
    thead: [checkbox all] | Row | Name | Email | Status
    tbody per row:
      [checkbox] | {row number} | {name} | {email} | [Badge variant]
      [if error] colspan full: text-sm text-danger "{error message}"

  [Summary bar — text-sm text-muted, gap-4]
    "{N} valid · {N} duplicates · {N} errors"

  [Footer]
    Button secondary "← Back"
    Button accent "Confirm import ({N} clients)"  ← count = checked valid rows
      [Loader2 spinner shown during import submission]
```

### Step 4: Result

```
MobileSheetContent
  MobileSheetHeader (no description)
    title: "Import complete"

  [Result summary — centered, py-8]
    Check icon (size-10, text-success, bg-emerald-50 rounded-full p-2)
    Heading text-base font-semibold: "{N} clients imported"
    [Grid 3-col — gap-md]
      "{N} imported"   text-success
      "{N} skipped"    text-muted   (duplicate emails)
      "{N} failed"     text-danger  (validation failures, normally 0)

  [Failed rows download — shown only when failed > 0]
    Button ghost with Download icon: "Download failed rows"

  [Footer]
    Button accent "Done"  ← closes sheet, triggers revalidatePath
```

## Interaction States

| Element | State | Visual Treatment |
|---------|-------|-----------------|
| File drop zone | Default | Dashed `border-border`, `bg-surface-subtle` |
| File drop zone | Drag over | `border-accent` border, `bg-accent-soft` background |
| File drop zone | File loaded | Solid `border-border-brand`, filename shown, `bg-surface` |
| Preview table row | Valid | Left border `border-l-2 border-success`, `bg-surface` |
| Preview table row | Duplicate | Left border `border-l-2 border-amber-400`, `bg-amber-50/40` |
| Preview table row | Error | Left border `border-l-2 border-danger`, `bg-red-50/40` |
| Preview table row | Unchecked | Opacity 50%, checkbox unchecked |
| Confirm button | Loading | `Loader2` spinner (animate-spin), button disabled |
| Select (mapping) | Auto-mapped | Shows detected value, "Auto-detected" subtext in muted |
| Select (mapping) | Unset required | Shows placeholder "Select column…" |

Step indicator (above wizard content):
- 4 numbered dots, connected by a line
- Current step: filled `bg-accent` circle, `text-on-dark`
- Completed steps: filled `bg-success` circle with Check icon, `text-on-dark`
- Upcoming steps: `bg-surface-strong` circle, `text-muted`

## Responsive Behavior

| Breakpoint | Wizard container | Preview table |
|------------|-----------------|---------------|
| Mobile (<1024px) | Vaul bottom drawer, full width | Horizontal scroll, col headers sticky |
| Desktop (≥1024px) | Centered Dialog, max-w-2xl (Steps 1–2, 4), max-w-3xl (Step 3) | Full table visible |

Mapping grid (Step 2): `grid-cols-1 sm:grid-cols-2` — two fields per row on ≥640px.

Touch targets: All checkboxes in the preview table use `size-11` (44px) hit area via wrapper. Select dropdowns use `h-12` default from existing `Select` component.

## Accessibility

- Wizard steps are announced via `aria-live="polite"` region when step changes
- File input has `aria-label="Upload CSV file"`
- Preview table uses `<table>` with proper `<thead>` / `<tbody>` / `<th scope="col">` markup
- "Select column" dropdowns have associated `<Label>` elements with `htmlFor`
- Drag-over state announced via `aria-describedby` on drop zone
- Focus is moved to the first interactive element when a step transitions
- Loader state: `aria-busy="true"` on the confirm button during submission
- Close (X) button has `aria-label="Close import wizard"`
