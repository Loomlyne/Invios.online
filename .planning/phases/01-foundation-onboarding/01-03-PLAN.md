```typescript
test("sign-out clears session and redirects to sign-in", async ({ page }) => {
  const credentials = await createConfirmedUser();
  await signIn(page, credentials);

  // Verify we're in the app
  await expect(page).toHaveURL(/\/app$/);

  // Click sign-out button
  await page.getByRole("button", { name: "Sign out" }).click();

  // Should redirect to sign-in
  await page.waitForURL(/\/sign-in/, { timeout: 10_000 });

  // Verify session is cleared — attempting to navigate to /app should redirect back to sign-in
  await page.goto("/app");
  await page.waitForURL(/\/sign-in/, { timeout: 10_000 });
});
```

This test:
1. Creates a confirmed user via admin API (same pattern as other tests).
2. Signs in using the existing `signIn` helper.
3. Clicks the "Sign out" button (rendered by `SignOutButton` component).
4. Asserts redirect to `/sign-in`.
5. Attempts to revisit `/app` — asserts middleware redirects back to `/sign-in` (session cleared).

NOTE: The `SignOutButton` component uses a client-side `signOutAction` which returns `{ redirectTo: "/sign-in" }`. The component reads `redirectTo` and calls `router.push`. The button text is "Sign out" (visible in the component). The button is always visible in the app header.
