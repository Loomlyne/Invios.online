Test cases:
- `describe("updateSession")`:
  - `it("passes through when Supabase is not configured")` — mock `isSupabaseConfigured` to return `false`, assert response has no redirect.
  - `it("redirects unauthenticated /app requests to /sign-in")` — mock user as null, request pathname `/app/invoices`, assert redirect location is `/sign-in` with `?next=/app/invoices` query param.
  - `it("redirects unauthenticated /app root to /sign-in")` — pathname `/app`, assert redirect to `/sign-in?next=/app`.
  - `it("redirects authenticated /sign-in requests to /app")` — mock user as present, pathname `/sign-in`, assert redirect to `/app`.
  - `it("redirects authenticated /sign-up requests to /app")` — mock user as present, pathname `/sign-up`, assert redirect to `/app`.
  - `it("passes through authenticated /app requests")` — mock user as present, pathname `/app/settings`, assert no redirect.
  - `it("passes through unauthenticated public routes")` — mock user as null, pathname `/`, assert no redirect.

Use `vi.mock()` for module mocking. The mock for `createServerClient` should return a function that returns the fake supabase client. The mock for `NextResponse` should use `vi.fn()` that returns objects with `.headers` and `.cookies` stubs.

NOTE: Because `updateSession` internally constructs `NextResponse` objects and calls `request.nextUrl.clone()`, the mocks must support:
- `request.nextUrl.pathname` (string)
- `request.nextUrl.clone()` returning an object with mutable `pathname` and `searchParams`
- `request.cookies.getAll()` returning `[]`
- `NextResponse.next()` returning `{ request, cookies: { set: vi.fn() } }`
- `NextResponse.redirect()` returning `{ headers: new Map(), cookies: { set: vi.fn() } }` with the redirect URL accessible

The simplest approach: create a `createMockRequest(pathname: string)` helper that builds the request stub, and mock `next/server` module at the top of the test file.
