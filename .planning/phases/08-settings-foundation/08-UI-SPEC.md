| sm | 8px | Compact element spacing (`gap-2`, `p-2`) |
| md | 16px | Default element spacing (`gap-4`, `px-4`) |
| lg | 24px | Section padding (`gap-6`, `p-6`) |

| 2xl | 48px | Page-level vertical rhythm |

Fluid tokens (from globals.css — use for layout, not fixed values):

| Token | Range | Usage |

## Sidebar Design Contract

### Desktop Sidebar

- Width: 224px (`w-56`), fixed, always visible at `lg+` breakpoints
- Background: `glass-panel` class + `border-r border-black/8` (separates from content area)
- Padding: `p-4` outer, nav items `px-4 py-3`
- Nav item inactive: `text-sm font-medium text-muted-strong`, icon `text-muted`, hover `bg-black/5`
- Nav item active: `bg-accent/10 text-accent-strong`, icon `text-accent`, `rounded-[var(--radius-md)]`
- Separator between Emails and Integrations: `<hr className="my-2 border-t border-border" />`
- ARIA: `<nav role="navigation" aria-label="Settings navigation">`, `aria-current="page"` on active button, roving `tabIndex` (active=0, inactive=-1)
- Section order: Profile → Branding → Business Info → General → Emails → [separator] → Integrations → Billing

Source: D-01, D-02, D-03, D-06 resolved; app-sidebar-nav.tsx active state pattern adapted

### Mobile Section Picker

- Trigger: Full-width `<button>` above content area, `lg:hidden`. Shows current section icon + label + ChevronDown
- Trigger classes: `w-full flex items-center justify-between px-4 py-3 rounded-[var(--radius-md)] border border-border bg-surface text-sm font-medium`
- Picker: Vaul `<Drawer>` — bottom sheet pattern (consistent with FAB popup precedent in project)
- Drawer items: icon (size-4) + text-sm label, matching desktop sidebar items exactly (D-05)
- Drawer background: `bg-surface` with `rounded-t-[var(--radius-lg)]`

Source: D-04 resolved (Vaul Drawer over dropdown select — icons requirement), RESEARCH.md Pattern 5

## Save Button Design Contract

States (D-09 locked):

| State | Label | Icon | Button state |
|-------|-------|------|-------------|
| idle, clean | "Save Changes" | none | `disabled` (grayed) |
| idle, dirty | "Save Changes" | none | enabled |
| saving | "Saving..." | `Loader2` spinning | `disabled` |
| saved | "Saved" | `Check` | `disabled`, 2s then resets to idle |

- Placement (D-08 resolved): In section card header, right-aligned, on desktop. On mobile: sticky at bottom of viewport (`sticky bottom-0 bg-surface border-t border-border px-4 py-3`), `w-full` button
- Accessibility: `<span role="status" aria-live="polite" className="sr-only">` announces "Changes saved." when state transitions to saved
- No toast, no banner. Inline state change only (D-09 locked)

Source: D-08 resolved (header on desktop, sticky bottom on mobile), D-09 locked

## Interaction States

### Sidebar Navigation (NAV-01 through NAV-05)

| Interaction | Behavior |
|-------------|----------|
| Click sidebar item | Sets `activeSection`, calls `router.replace('/app/settings?section={section}', { scroll: false })` |
| Browser Back/Forward | URL-driven; `initialSection` re-read from `searchParams` on server render |
| Arrow Down on sidebar item | Focus moves to next item (roving tabindex `onKeyDown` handler) |
| Arrow Up on sidebar item | Focus moves to previous item |
| Tab into sidebar | Focus lands on active item (`tabIndex={0}`); all others `tabIndex={-1}` |
| Focus ring | `outline: 2px solid var(--accent); outline-offset: 3px` — global `:focus-visible` |

### Section Placeholder Panels (Phase 8 scope)

- Each placeholder renders: section title (text-lg semibold), section description (text-sm muted), and a disabled `<SaveButton isDirty={false} />` in the card header
- No wired actions in Phase 8 — Save button disabled with `isDirty={false}`
- Integrations and Billing panels render a "Coming soon" card with muted text, no Save button

## Layout Architecture

```
/app/settings page (Server Component)
  └── SettingsShell (Client Component)
        ├── [lg+ only] SettingsSidebar (224px fixed)
        │     ├── Nav item: Profile (User icon)
        │     ├── Nav item: Branding (Palette icon)
        │     ├── Nav item: Business Info (Building2 icon)
        │     ├── Nav item: General (SlidersHorizontal icon)
        │     ├── Nav item: Emails (Mail icon)
        │     ├── [separator]
        │     ├── Nav item: Integrations (Plug icon)
        │     └── Nav item: Billing (CreditCard icon)
        └── Main content area (flex-1)
              ├── [mobile only] Vaul Drawer trigger (section picker)
              └── Active panel (one of 7 placeholder panels)
                    └── Card (bg-white, border)
                          ├── CardHeader (panel title + SaveButton right-aligned)
                          └── CardContent (placeholder or "coming soon")
```

Responsive breakpoint: `lg` (1024px) is the only breakpoint that controls sidebar visibility. Below `lg`, sidebar is `hidden`, mobile picker is visible.

Bottom padding: `pb-28 lg:pb-8` on the content area to account for mobile bottom nav (existing project pattern).
