**Function: buildRevenueTrend**
```typescript
export function buildRevenueTrend(
  rows: DashboardInvoiceRow[],
  payments: PaymentRecord[],
  today: string,
): RevenueTrendMonth[] {
```
Implementation rules:
- Generate 12 month slots going backward from `today`. Use `new Date(\`${today}T00:00:00\`)`, then loop `i = 11` down to `0`, calling `d.setMonth(d.getMonth() - i)` on a fresh clone each iteration, producing `d.toISOString().slice(0, 7)` as the monthKey.
- For each monthKey, `billed` = sum of `row.total` where `row.issueDate.slice(0, 7) === monthKey` (use `toDateOnly(row.issueDate)?.slice(0, 7)`). Only include non-draft rows (rows are already non-draft from `buildDashboardInvoiceRows`).
- For each monthKey, `collected` = sum of `payment.amount` where `payment.datePaid.slice(0, 7) === monthKey`. This uses raw PaymentRecord[] â€” NOT DashboardInvoiceRow.collectedAmount â€” to correctly attribute payments to the month they were made, not the invoice issue month.
- The `month` display label: use `["Jan","Feb","Mar","Apr","May","Jun","Jul","Aug","Sep","Oct","Nov","Dec"][parseInt(monthKey.slice(5, 7), 10) - 1]`.
- Wrap all sums with `roundCurrency()`.
- Always return exactly 12 entries, zero-padded.

**Function: buildAgingBuckets**
```typescript
export function buildAgingBuckets(
  rows: DashboardInvoiceRow[],
  today: string,
): AgingBucket[] {
```
Implementation rules:
- Filter to rows with `outstandingAmount > 0` (excludes fully paid invoices).
- For each remaining row, compute `daysOverdue = dayDifference(row.dueDate, today)`. If `daysOverdue` is null, skip the row. Use `Math.max(0, daysOverdue)` so not-yet-due invoices land in the 0-30 bucket.
- Bucket: 0-30 (daysOverdue 0..30), 31-60 (31..60), 61-90 (61..90), 90+ (>90).
- Always return exactly 4 buckets in order, even if some have amount: 0, count: 0.
- Use `roundCurrency()` for amount sums.

**Function: buildMomDeltas**
```typescript
export function buildMomDeltas(
  rows: DashboardInvoiceRow[],
  range: DashboardRangeKey,
  today: string,
): MomDeltas {
```
Implementation rules:
- If `range === "all"`, return `{ totalBilled: null, totalCollected: null, outstanding: null, collectionRate: null }`.
- Compute the current period window: `rangeStart = buildRangeStart(range, today)`, `rangeEnd = today`.
- Compute the prior period window: shift both rangeStart and rangeEnd backward by the same interval. For "30d": 30 days back. For "90d": 90 days back. For "12m": 12 months back. Use the same `new Date` + setDate/setMonth pattern as `buildRangeStart`.
- Filter `rows` to current period (where `toDateOnly(row.issueDate)` falls within `[currentStart, currentEnd]`) and build metrics via `buildDashboardMetrics` on a temporary set of rows with `issuedInRange: true`, `collectedInRangeAmount` recalculated.
  - **Simpler approach:** Create a helper that re-runs `buildDashboardInvoiceRows`-style range filtering. Actually, since `rows` are from the full dataset (all-time), we can filter by checking `toDateOnly(row.issueDate) >= periodStart && toDateOnly(row.issueDate) <= periodEnd`.
  - For each period, compute: `totalBilled = sum(filteredRows.map(r => r.total))`, `totalCollected = sum(filteredRows.map(r => r.collectedAmount))` (all-time collected for invoices issued in that period), `outstanding = sum(filteredRows.map(r => r.outstandingAmount))`, `collectionRate = computeCollectionRate({ totalBilled, totalCollected })`.
- For each metric, delta = `((current - prior) / |prior|) * 100`. If prior === 0, return null for that metric.
- For `collectionRate`: delta = `currentRate - priorRate` (percentage-point difference, per D-09). If either rate is null, return null.
- Round all deltas to 1 decimal place via `Math.round(delta * 10) / 10`.
