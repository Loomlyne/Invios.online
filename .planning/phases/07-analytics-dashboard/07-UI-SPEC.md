## Spacing Scale

All spacing tokens are multiples of 4. Fluid tokens from `globals.css` take precedence over fixed values inside dashboard layout containers.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, pill badge internal padding |
| sm | 8px | Between label and value in MetricCard badge |
| md | 16px | Default element padding (`p-4` on MetricCard, insight strip cells) |
| lg | 24px | Section gaps (`--space-section` clamps 16–24px) |
| xl | 32px | Chart container internal padding |
| 2xl | 48px | Major section vertical separation |
| 3xl | 64px | Page-level outer spacing |

Fluid tokens (declared in `:root` — source: globals.css):
- `--space-section`: clamp(1rem, 0.5rem + 1.25vw, 1.5rem) — vertical gaps between dashboard sections
- `--space-card`: clamp(1rem, 0.625rem + 0.9375vw, 1.5rem) — Card internal padding
- `--space-grid`: clamp(0.75rem, 0.5rem + 0.625vw, 1rem) — gaps inside grid rows (MetricCards, insight strip, chart row)
- `--radius-card`: clamp(1rem, 0.6rem + 1vw, 1.6rem) — Card border-radius
- `--radius-inner`: clamp(0.75rem, 0.45rem + 0.75vw, 1.1rem) — inner element border-radius (MetricCard, insight strip cells, chart wrapper)

Exceptions: Touch targets on interactive MetricCards have `min-h-24` (96px) on mobile, `min-h-28` (112px) on sm+. MoM badge pill: 4px vertical / 8px horizontal internal padding.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (text-sm) | 400 (regular) | 1.5 | Chart axis labels, table cells, badge text, description copy |
| Label | 12px (text-xs) | 400 (regular) / 600 (semibold) | 1.4 | Section eyebrows (uppercase, tracking-[0.2em]), metric card labels (tracking-[0.18em]), chart legend labels |
| Heading | 20px (text-xl) | 600 (semibold) | 1.2 | CardTitle (existing: `text-xl font-semibold tracking-tight`) |
| Value | clamp(18px, 14px + 0.625vw, 24px) | 600 (semibold) | 1.1 | MetricCard value (existing: `text-[clamp(1.125rem,0.875rem+0.625vw,1.5rem)]`) |

Only two weights in use: **400** (regular) and **600** (semibold). No other weights.

Tabular numerics: apply `tabular-nums` class to all currency values and percentage values — existing pattern on MetricCard value, carry through to MoM badge and chart labels.

## Color

All values sourced from `src/app/globals.css` `:root` block. No new color tokens are introduced in this phase.

| Role | CSS Var / Value | Usage |
|------|-----------------|-------|
| Dominant (60%) | `--background` #f8f4ee | Page background, body gradient base |
| Secondary (30%) | `--surface` #fffdf9 / `#FFF8EE` / `#FFF7EA` | Card backgrounds, insight strip cells, table rows, chart card wrapper |
| Accent (10%) | `--accent` #ca8a04 | See reserved list below |
| Success | `--success` #24583a | MoM positive badge text and background tint |
| Danger | `--danger` #8d3d2e | MoM negative badge text; aging 90+ day segment |
| Muted | `--muted` #6b6359 | Axis labels, legend text, card descriptions |
| Muted strong | `--muted-strong` #3a322c | Secondary data in table cells |
| Border | `--border` rgba(28,25,23,0.12) | Card and cell borders |
| Border brand | `--border-brand` #d7c4a7 | Active MetricCard border, hover state borders |
| Accent soft | `--accent-soft` rgba(202,138,4,0.12) | MoM neutral/null badge background (when no data) — not used |
| Surface strong | `--surface-strong` #f1e9dc | Chart card header backgrounds, aging segment: 0-30 days bucket |

**Accent (#ca8a04) reserved for:**
- Focus ring (`outline: 2px solid var(--accent)`) — already global
- Active MetricCard dot indicator
- "New invoice" button (variant="accent")
- Chart bar: "Collected" bar (positive signal — money received)

**Chart color palette** (derived from HSL token family — no new hex values):

| Segment | Color | CSS / Hex |
|---------|-------|-----------|
| Revenue trend — Billed bar | `--muted-strong` / #3a322c at 60% opacity | Neutral warm dark, represents issued amount |
| Revenue trend — Collected bar | `--accent` #ca8a04 | Accent gold — money in hand |
| Aging — 0–30 days | `--surface-strong` #f1e9dc with `--muted-strong` text | Least urgent, warm neutral |
| Aging — 31–60 days | `rgba(202,138,4,0.28)` (accent-glow level) | Mild urgency, warm amber tint |
| Aging — 61–90 days | `rgba(202,138,4,0.55)` | Elevated urgency, denser amber |
| Aging — 90+ days | `--danger` #8d3d2e | Overdue — semantic red |

**MoM badge colors:**
- Positive delta (green pill): background `rgba(36,88,58,0.10)`, text `--success` #24583a, arrow icon TrendingUp
- Negative delta (red pill): background `rgba(141,61,46,0.10)`, text `--danger` #8d3d2e, arrow icon TrendingDown
- Null delta (no prior data): badge hidden entirely (D-07 decision)

**Collection rate MoM format** (D-09, Claude's discretion): Display as percentage-point delta with "pp" suffix — e.g., "+3pp" or "-5pp". Rationale: the card already shows an absolute percentage (e.g., "84%"), so showing "+3%" would be ambiguous (does it mean 84% → 87% or 84% → 84% × 1.03?). "+3pp" is unambiguous and familiar in finance contexts.

### Chart Card Internal Layout

Each chart is wrapped in a shadcn `Card` with:
- `CardHeader`: title + description (no action button needed)
- `CardContent`: chart component loaded via `next/dynamic`
- Minimum height: 240px on mobile, 280px on md+
- Chart fills 100% of card width minus `--space-card` padding

### MetricCard MoM Badge Position

Badge sits between the metric value line and the "Click to inspect" hint line (for interactive cards). For non-interactive cards, badge sits directly below the value. Badge is inline-flex, pill-shaped, `rounded-full`, 12px text, gap-1 between arrow icon and delta text.

## Interaction Contract

### MetricCard with MoM Badge

- Badge is purely informational — no click target
- Arrow icon: `TrendingUp` (lucide) for positive, `TrendingDown` (lucide) for negative
- Badge size: 12px text, icon size-3 (12px), px-2 py-0.5, rounded-full
- No animation on badge — static display
- Transition: inherits parent MetricCard transition on hover

### Revenue Trend Chart

- Tooltip on hover: shows month label, billed AED, collected AED
- Legend: two entries ("Billed" / "Collected") using the defined palette
- X-axis: 3-letter month abbreviations (Jan, Feb, … Dec)
- Y-axis: currency values, formatted as abbreviated AED (e.g., "5K", "12K") — no full `formatCurrency` in axis labels
- No click interaction on bars — read-only visualization
- Responsive: `<ResponsiveContainer width="100%" height={260}>`
- Reduced motion: Recharts respects `prefers-reduced-motion` — no additional handling needed

### Aging Breakdown Chart

- Horizontal stacked bar (single bar across full width)
- Each segment labeled with: amount (AED abbreviated) + count (e.g., "3 inv")
- Labels hidden when segment width is too narrow (< 60px) to avoid overflow
- Tooltip on hover: shows bucket label, AED total, invoice count
- No click interaction — read-only
- Responsive: `<ResponsiveContainer width="100%" height={120}>` (compact, fits ~40% column)

### Chart Empty State

When zero non-draft invoices exist (SC-4 from requirements):
- Render `<EmptyState>` in place of the chart
- Title: "No revenue data yet."
- Description: "Issue your first invoice to start tracking trends."
- No action button inside the chart card empty state (the PageHeader already has "New invoice")

## Accessibility Contract

- All chart containers: `role="img"` with `aria-label` describing the chart purpose
- MoM badge arrow icons: `aria-hidden="true"` — the delta number is the accessible text
- MoM badge container: no `aria-label` needed (value is readable as plain text)
- Color is never the sole differentiator — aging segments also differ by position and label
- Focus ring: inherits global `outline: 2px solid var(--accent)` on `:focus-visible`
- Chart tooltips: keyboard-accessible via Recharts built-in (tab to chart, arrow keys navigate)
