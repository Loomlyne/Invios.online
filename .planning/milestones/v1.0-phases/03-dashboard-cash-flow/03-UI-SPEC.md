## Spacing Scale (inherited)

Standard 8-point scale using Tailwind's default spacing (4px base unit).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`, `p-1`) | Icon gaps, ledger row cell padding, delete icon button padding |
| sm | 8px (`gap-2`, `p-2`) | Inline table row gaps, badge gap |
| md | 16px (`gap-4`, `p-4`) | Card internal padding, form field gaps, ledger row horizontal padding |
| lg | 24px (`gap-6`, `p-6`) | Section gaps, card padding default |
| xl | 32px (`gap-8`) | Layout gaps between dashboard sections |
| 2xl | 48px | Major dashboard section breaks |
| 3xl | 64px | ‚Äî not used in Phase 3 |

Exceptions for Phase 3:
- Inline ledger table rows: `px-4 py-3` (16px/12px) ‚Äî tighter than standard card padding; matches existing setup checklist item pattern
- Metric strip / stat cards: `p-4 sm:p-5` ‚Äî compact to fit 4 metrics in a row on desktop
- Touch targets for delete row action (icon button): minimum `size-8` (32px) with `p-1` (4px) ‚Äî smaller than standard `size-10` because it sits inside a dense table row; acceptable within inline data context
- Mobile: `pb-28` main column bottom padding carries over from Phase 1 app shell

## State Feedback Patterns (Phase 3 extensions)

Inherits all Phase 1 state patterns. Phase 3 additions:

### Inline row save
- Optimistic: row appears immediately with `opacity-60 pointer-events-none` while server action runs
- Success: row renders at full opacity, add row resets to empty
- Error: inline `text-xs text-danger` below the add row ‚Äî "[What failed]. Try again."

### Status badge auto-update
- `DocumentStatusBadge` component already handles `partial_paid`, `paid`, `overdue` variants
- `overdue` ‚Üí maps to Badge `variant="warning"` (amber) ‚Äî matches Phase 2 implementation
- `partial_paid` ‚Üí maps to Badge `variant="default"` (neutral) with label "Partial Paid"
- No loading state shown during automatic status recompute ‚Äî it runs server-side on load

### Overdue count indicator
- Dashboard overdue section card header shows `Badge variant="warning"` with overdue count, e.g. "Overdue (3)"
- No separate notification or alert ‚Äî the section card is the indicator

### Empty state ‚Äî overdue section
- Not shown (section hidden when no overdue items). Source: CONTEXT.md D-07 (locked)

### Empty state ‚Äî recent documents columns
- `EmptyState` component, no icon variant (icon container hidden), centered text
- Heading: "No invoices yet" / "No quotations yet"
- Body: "Create your first invoice to start tracking." / "Create your first quotation to start tracking."

### Empty state ‚Äî payments table
- `<p className="px-4 py-5 text-sm text-muted text-center">No payments recorded yet.</p>` ‚Äî inline, not full EmptyState

### Empty state ‚Äî expenses table
- `<p className="px-4 py-5 text-sm text-muted text-center">No expenses recorded yet.</p>` ‚Äî inline, not full EmptyState

### New user dashboard (DASH-04)
- Shows existing SetupChecklist floating panel (already implemented)
- Metric strip still renders but all values show as "‚Äî" (em dash) when no invoice data exists
- Quick actions strip always renders ‚Äî it's the primary CTA for new users
- Recent documents columns show empty state copy above

Source: REQUIREMENTS.md DASH-04, Phase 3 CONTEXT.md

## Interaction Contracts

### Add row ‚Äî field validation

- Amount field: must be numeric and > 0. Show `text-xs text-danger` inline below field on submit attempt with non-numeric or zero value: "Enter a valid amount."
- Date field: must be non-empty. Error: "Enter a date."
- Description (expenses): must be non-empty. Error: "Enter a description."
- Method (payments): optional ‚Äî defaults to "other" if blank.
- Vendor (expenses): optional ‚Äî blank is acceptable.

### Status recompute trigger

After every payment or expense mutation (add/delete), the server action calls the status computation function. The invoice detail page revalidates via `revalidatePath`. The status badge in the page header updates on the next render ‚Äî no client-side state needed.

Source: CONTEXT.md D-03, D-09 (locked)

### Collection rate display

- Formula: `Math.round((totalCollected / totalBilled) * 100)` ‚Üí integer string + "%"
- When `totalBilled === 0`: display "‚Äî" (em dash) ‚Äî not "0%" to avoid misleading division-by-zero
- When 100%: render value with `text-success` color

Source: CONTEXT.md D-06, discretion applied for zero-state

### Overdue recompute trigger

On every dashboard load and invoices list load: server-side function compares `due_date` to today's date, updates status to `overdue` for unpaid/partial_paid invoices past due. Written back to `invoices` table. No UI trigger needed.

Source: CONTEXT.md D-08 (locked)
