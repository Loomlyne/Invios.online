**Component 3: `src/components/public/accept-reject-form.tsx`** (`"use client"`)

Props:
```typescript
interface AcceptRejectFormProps {
  shareToken: string;
  currentStatus: string;
}
```

This is a client component using `useActionState` from React for form state management.

Logic:
1. Import `acceptQuotationPublicAction` and `rejectQuotationPublicAction` from `@/actions/public-quotations`.
2. Use `.bind(null, shareToken)` to create bound versions of both actions (shareToken as first arg per the action signature).
3. State: `const [acceptState, acceptAction, acceptPending] = useActionState(boundAccept, { status: "idle" })` and same for reject.
4. Local state: `const [showRejectNote, setShowRejectNote] = useState(false)` — controls textarea visibility.

Rendering per UI-SPEC Interaction Contract — Public Quotation Page:
- If `currentStatus` is `"accepted"`: show `<p className="text-sm text-success font-medium">This quotation has been accepted.</p>`. No buttons.
- If `currentStatus` is `"rejected"`: show `<p className="text-sm text-danger font-medium">This quotation has been rejected.</p>`. No buttons.
- If `acceptState.status === "success"`: show success message "Quotation accepted — the sender has been notified." in `text-success`.
- If reject state is success: show "Quotation rejected. Your note has been sent." in `text-danger`.
- Otherwise (status is `"sent"`): render Accept and Reject buttons.

Accept button: `<form action={acceptAction}>` wrapping `<Button type="submit" disabled={acceptPending} className="h-11 min-w-[160px]" style={{ backgroundColor: "var(--accent)" }}>Accept Quotation</Button>`.

Reject flow (two-step per D-04, UI-SPEC):
- First click: `setShowRejectNote(true)`. Button label: "Reject".
- After click: textarea slides in with CSS transition (`transition-all duration-200 ease-out`). Textarea placeholder: "Add a note for the sender (optional)". Below textarea: "Confirm Rejection" button in danger fill.
- The reject form: `<form action={rejectAction}>` wrapping the `<Textarea name="rejectionReason" />` and the confirm button. Do NOT include a hidden input for rejectionReason — the Textarea alone is sufficient. `formData.get("rejectionReason")` will return the textarea value directly, or an empty string if the user leaves it blank.

Layout: Accept and Reject side-by-side on desktop (`flex gap-3`), stacked on mobile (`flex-col`). Accept: accent fill. Reject: outline variant with danger border (`variant="outline" className="border-danger text-danger"`). Confirm Rejection: `variant="destructive"` or `className="bg-danger text-on-dark"`.

Error state: if `acceptState.status === "error"` or reject error, show inline error message: "Something went wrong. Please try again or contact the sender." in `text-danger text-sm`.
