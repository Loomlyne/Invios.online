This is the UX-03 comprehensive visual quality pass. The executor must READ every file listed above, identify the highest-impact inconsistencies, and fix them. The goal is to make every view feel like it belongs to the same premium product — not to redesign anything, but to polish and unify.

**Audit checklist — for each file, check and fix:**

1. **Spacing consistency**: Replace arbitrary pixel values with consistent 4px-increment spacing. Use `gap-4` (16px), `gap-6` (24px), `py-4`, `px-6` consistently. Fix any `mt-3`, `gap-[21px]`, or unusual spacing values that break the 4px grid. The dashboard page (`page.tsx`) is known to have `gap-[21px]` on line 57 — normalize to `gap-5` (20px) or `gap-6` (24px).

2. **Border radius consistency**: Use `rounded-[var(--radius-lg)]` (1.75rem), `rounded-[var(--radius-md)]` (1.1rem), or `rounded-[var(--radius-sm)]` (0.85rem) from globals.css. Replace any hardcoded `rounded-lg`, `rounded-xl`, `rounded-2xl` that don't match the design tokens.

3. **Color consistency**: Replace any hardcoded hex colors that duplicate design tokens:
   - `#78716C` → `text-muted` (close to --muted #6b6359, but check if intentional)
   - `#A8A29E` → consider a lighter muted variant
   - `#FFF8EE` → `bg-surface` (close to --surface #fffdf9)
   - Ensure no accent color (#ca8a04) used outside the reserved accent contexts per UI-SPEC

4. **Typography alignment**: Ensure headings use consistent size/weight per UI-SPEC typography scale. Page titles should be 20px/600 (`text-xl font-semibold`), section labels 14px/400 (`text-sm`), body 16px/400 (`text-base`).

5. **Card elevation**: All Card components should use consistent border treatment: `border border-black/8` (matches InvoicePreview) rather than mixed border styles.

6. **Status badge consistency**: Ensure `DocumentStatusBadge` is used everywhere statuses appear — no inline badge rendering with different styling.

7. **Empty state consistency**: Check that all empty states (no invoices, no quotations, no clients) use consistent styling: centered, icon/illustration area, heading + body text in muted color.

8. **Mobile touch targets**: Buttons should be minimum 44px tall on mobile. Check detail page action buttons, list page action buttons.

9. **Table/list hover states**: Ensure interactive rows have `hover:bg-surface-strong/50` or equivalent subtle hover feedback.

10. **Dashboard MetricCard polish**: Review `metric-card.tsx` for consistent padding, border, and number formatting. Ensure numbers use `tabular-nums` for alignment.

**Specific known issues to fix:**

- Dashboard `page.tsx` line ~57: `gap-[21px]` — normalize to `gap-5` or `gap-6`
- Invoice detail `page.tsx`: `style={{ height: 26 }}` hardcoded on CardTitle — remove, use natural line height
- Quotation detail InfoCard: uses `bg-[#FFF8EE]` — normalize to `bg-surface` for consistency
- Any `style={{ height: ... }}` inline overrides should be replaced with Tailwind classes

**CSS additions if needed** in `globals.css`:
- If a commonly needed utility is missing (e.g., a glass-panel hover state), add it as a utility class rather than repeating inline styles

The executor should systematically open each file, audit against the checklist, make changes, and move to the next file. This is a polish pass — changes should be styling-only. No functional logic changes.
