**Synthesized:** 2026-06-10  
**Sources:** v2.0-FEATURES, v2.0-ARCHITECTURE, v2.0-STACK, v2.0-PITFALLS, codebase audit, user milestone context

---

## Executive Summary

v2.0 transforms Invios from a complete billing MVP into an **operator power platform** by shipping eight user-requested capability areas across **eight phases (13–20)**, after completing **v1.2 Settings (phases 9–12)**. Stripe and native payment collection are explicitly excluded.

The highest-leverage early win is **Phase 13: Client Portal v2** — human-readable URLs (`/portal/[operatorSlug]/[clientPortalSlug]`), full document history, and branded experience. This directly satisfies the user's portal request and reuses existing `clients.slug`, branding tokens, and portal page infrastructure.

## Key Architectural Decisions

1. **Portal URL:** `/portal/[operatorSlug]/[portalSlug]` with per-user uniqueness
2. **Intelligence:** Computed services, not new CRM tables
3. **Automation:** JSONB rules + cron evaluator, fixed action vocabulary
4. **AI:** Vercel AI SDK + Zod; draft-only, never auto-send
5. **Integrations:** Outbound webhooks first; Zapier via webhook URL
