# Feature Research: v2.0 Operator Power

**Domain:** Premium invoicing operator console — power addons for Invios  
**Researched:** 2026-06-10  
**Confidence:** HIGH (builds on shipped v1.0–v1.1 codebase; patterns from HoneyBook, FreshBooks, Linear)

---

## Feature Landscape

### Table Stakes (Users Expect These)

| Feature | Why Expected | Complexity | Notes |
|---------|--------------|------------|-------|
| Named client portal URL | Clients bookmark `/portal/their-company`; opaque tokens feel untrustworthy | MEDIUM | `clients.slug` exists; add `portal_slug` + route `/portal/[portalSlug]` |
| Portal document history | v1.0 portal lists invoices/quotes only — users want status, totals, overdue visibility | LOW | Extend existing `portal/[portalToken]/page.tsx` |
| Time entries → invoice lines | Every freelancer tool (Toggl, Harvest, FreshBooks) connects time to billing | MEDIUM | New `time_entries` table; uses `profiles.hourly_rate` from v1.2 Phase 10 |
| Client payment reliability | Operators ask "who pays late?" before sending more work | MEDIUM | Compute from `payments` + `invoices.due_date` — no new user input |
| Rule-based automation | "When quote accepted, create invoice" is standard in HoneyBook/FreshBooks | HIGH | New `automation_rules` JSONB or normalized tables + cron evaluator |
| Cash runway visibility | Solo operators need "will I have enough cash in 60 days?" | MEDIUM | Projection from recurring schedules + aging + historical collection rate |
| Proposal approval trail | Legal/commercial need: who accepted, when | MEDIUM | Extend `quotations` with `accepted_at`, `accepted_by`, signature capture optional |
| Webhook/export integrations | Power users want Zapier/Make or custom webhooks | MEDIUM | Outbound webhooks on document events; CSV/API export hooks |

### Differentiators (Competitive Advantage)

| Feature | Value Proposition | Complexity | Notes |
|---------|-------------------|------------|-------|
| AI draft from pasted brief | Eliminates retyping client emails into line items | HIGH | Vercel AI SDK + structured output; guardrails for TRN/AED |
| AI invoice review | Catches TRN missing, math errors, negative margin before send | MEDIUM | Rule-based + optional LLM layer |
| Smart reminder suggestions | "Client X usually pays in 45 days — send reminder day 38" | MEDIUM | Uses client intelligence scores |
| Branded portal per operator | Portal inherits logo, colors from `ownerState.branding` | LOW | Already partial in v1.0 — extend |
| Bilingual proposal flow | UAE market: proposal in EN/AR with same approval mechanics | MEDIUM | Reuse Phase 4 bilingual patterns |
| Automation without Zapier tax | Built-in rules reduce need for external tools for common flows | HIGH | Key moat for solo operators |

### Anti-Features (Avoid This Milestone)

| Feature | Why Problematic | Alternative |
|---------|-----------------|-------------|
| Stripe / payment links | User has no Stripe account | Manual payment recording (existing); revisit in v3 |
| Full CRM pipeline | Scope creep — deals, pipelines, email sequences | Client intelligence cards only |
| Native e-sign legal compliance | DocuSign-level audit trail is a product | Lightweight click-to-accept + timestamp + IP log |
| Real-time collaborative editing | Solo operator product | Single-user edits with version history (existing) |
| Custom scripting in rules | Security + support burden | Fixed trigger/action vocabulary |

## Feature Dependencies

```
portal_slug + enhanced portal UI (PORT)
    └── client intelligence displays portal engagement (optional metric)

profiles.hourly_rate (PROF-06, Phase 10)
    └── time tracking billable rate (TIME)

Phase 5 recurring + reminder crons
    └── automation rules evaluator (AUTO)
    └── AUTO-06 auto-send recurring

Phase 7 analytics (trend, aging, MoM)
    └── cash flow forecast (FCST)
    └── DASH-08 cohort extension

Phase 4 public quotation accept/reject (PUB-05)
    └── proposals + approval workflow (PROP)

Phase 12 integrations placeholder
    └── integrations hub (INT) — webhooks, exports

All structured billing data
    └── AI co-pilot (AI) — last phase
```
