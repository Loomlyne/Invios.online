# Pitfalls Research: v2.0 Operator Power

**Researched:** 2026-06-10  
**Confidence:** HIGH (from v1.0 shipped lessons + domain patterns)

---

## Critical Pitfalls

### 1. Portal Slug Collisions Across Tenants

**What goes wrong:** `/portal/acme` resolves to wrong operator's client.  
**Why:** Slug only scoped to client name, not operator.  
**Prevention:** Unique index on `(user_id, portal_slug)`; public URL includes `operatorSlug` segment: `/portal/[operatorSlug]/[portalSlug]`.  
**Phase:** 13

### 2. Automation Rule Infinite Loops

**What goes wrong:** "Quote accepted → create invoice → triggers quote accepted" cascade.  
**Why:** Actions fire events that re-trigger rules.  
**Prevention:** Event idempotency keys; rules cannot trigger on self-generated events; max depth 1 per cron run.  
**Phase:** 16

### 3. AUTO-06 Auto-Send Without Review Guardrails

**What goes wrong:** Recurring invoice emails wrong client or duplicate sends.  
**Why:** Cron retries + missing dedup.  
**Prevention:** Extend Phase 5 `last_sent_at` dedup; dry-run preview in UI; operator enable per schedule.  
**Phase:** 16 (bundled with automation)

### 4. Time Entry Double-Billing

**What goes wrong:** Same time entry converted to multiple invoices.  
**Why:** Missing `invoiced_at` / `invoice_id` lock.  
**Prevention:** Atomic update on convert; UI shows "already billed" state.  
**Phase:** 15

### 5. Forecast False Precision

**What goes wrong:** User trusts 90-day number that assumes 100% collection.  
**Why:** Model ignores historical collection rate.  
**Prevention:** Show range (optimistic / expected / conservative); label assumptions in UI.  
**Phase:** 17

### 6. AI Hallucinated Line Items or Tax

**What goes wrong:** AI invents TRN or wrong AED amounts; user sends to client.  
**Why:** LLM not grounded in user settings.  
**Prevention:** Always show draft in editor before save; deterministic validation pass; never auto-send.  
**Phase:** 20

### 7. Webhook Secret Leakage

**What goes wrong:** Webhook signing secret exposed in client bundle.  
**Why:** Accidental client-side import.  
**Prevention:** Webhook dispatch server-only; secrets in DB encrypted at rest (future) or env per subscription.  
**Phase:** 19

### 8. Scope Explosion in Phase 20

**What goes wrong:** AI phase blocks ship of portal/time tracking.  
**Why:** Wrong phase ordering.  
**Prevention:** AI is Phase 20 last; each prior phase ships standalone value.  
**Phase:** Roadmap order

## v1.2 Completion Pitfalls (Phases 9–12)

### 9. Settings Refactor Breaks Existing Flows

**What goes wrong:** Monolithic settings split breaks onboarding defaults.  
**Prevention:** Feature-flag panels; keep server action contracts stable; E2E settings smoke test.

### 10. Billing Panel Implies Stripe

**What goes wrong:** Users expect payment when only display stub exists.  
**Prevention:** Copy: "Plan management — payment provider coming soon"; no Stripe env vars required.

## Recovery Strategies

| Pitfall | Detection | Rollback |
|---------|-----------|----------|
| Portal slug leak | E2E test cross-tenant access | Disable slug route; token-only |
| Automation loop | Cron log spike | Disable rule; kill switch column |
| AI bad draft | User report | Disable AI routes via env flag |
